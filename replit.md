# Atlantis Archive

## Overview

Atlantis Archive is a themed document management web application with a "Deep Ocean" aesthetic. Users authenticate via Discord OAuth2, and their Discord server roles from the Atlantis guild determine what content they can access. Admins (Leviathan Lord, Hydra Overlord, Guardian) can create, edit, and delete documents. Documents are organized by categories with role-based access control.

## User Preferences

Preferred communication style: Simple, everyday language.

## System Architecture

### Frontend
- **Framework**: React 18 with TypeScript, bundled by Vite
- **Routing**: Wouter (lightweight React router)
- **State Management**: TanStack React Query for server state caching and mutations
- **UI Components**: shadcn/ui (new-york style) built on Radix UI primitives
- **Styling**: Tailwind CSS with CSS variables for theming (deep ocean color palette: navy, teal, gold)
- **Animations**: Framer Motion for the landing page
- **Forms**: React Hook Form with Zod validation via @hookform/resolvers
- **Icons**: lucide-react for UI icons, react-icons/si for brand logos (Discord)
- **Path aliases**: `@/` maps to `client/src/`, `@shared/` maps to `shared/`

### Backend
- **Framework**: Express.js running on Node with TypeScript (tsx for dev, esbuild for production)
- **HTTP Server**: Node's built-in `http.createServer` wrapping Express
- **API Pattern**: RESTful JSON API under `/api/` prefix. Routes are defined in `server/routes.ts` using a shared API contract from `shared/routes.ts`
- **Shared API Contract**: `shared/routes.ts` defines route paths, HTTP methods, Zod input/output schemas — used by both client and server for type safety and validation
- **Dev Server**: Vite middleware serves the frontend in development with HMR
- **Production**: Client is built to `dist/public/`, server is bundled with esbuild to `dist/index.cjs`

### Database
- **Database**: PostgreSQL (required, referenced via `DATABASE_URL` environment variable)
- **ORM**: Drizzle ORM with `drizzle-zod` for schema-to-Zod conversion
- **Schema Location**: `shared/schema.ts` (documents, document versions, categories, role mappings) and `shared/models/auth.ts` (users, sessions)
- **Migrations**: Generated by `drizzle-kit` into `./migrations/` directory. Use `npm run db:push` to push schema changes
- **Key Tables**:
  - `users` — stores authenticated user profiles (id, email, name, profile image, discordId, discordUsername, discordAvatar, discordRoles[])
  - `sessions` — server-side session storage for express-session via connect-pg-simple
  - `documents` — main documents with title, content, category, author, visibility, allowedRoles[]
  - `document_versions` — version history for documents

### Authentication
- **Method**: Discord OAuth2
- **Implementation**: `server/discord-auth.ts`
  - OAuth2 flow: `/api/login` → Discord authorize → `/api/callback` → session created
  - Bot API: Fetches guild member roles from Atlantis Discord server using bot token
  - Session: express-session with connect-pg-simple, stored in the `sessions` table
  - `/api/auth/user` — get current user
  - `/api/auth/refresh-roles` — refresh Discord roles from server
  - `/api/auth/available-roles` — list all guild roles (for document role selector)
- **Environment Variables Required**: `DATABASE_URL`, `SESSION_SECRET`, `DISCORD_CLIENT_ID`, `DISCORD_CLIENT_SECRET`, `DISCORD_BOT_TOKEN`, `DISCORD_GUILD_ID`
- **Middleware**: `isAuthenticated` guard protects API routes, attaches user to request

### Role-Based Access Control
- **Admin Roles** (full access to everything): Leviathan Lord, Hydra Overlord, Guardian
- **Category Access**:
  - Guidelines, New Players, Military, Economy, First Steps → Admins + Tempest, Hydra, Atlantean
  - Internal Affairs → Admins + IA role
  - Foreign Affairs → Admins + FA role
  - Technology → Admins + Tech role
  - Intelligence → Admins only
- **Per-Document Restrictions**: Documents can have an `allowedRoles` array. If set, only users with those specific roles (plus admins) can see the document.
- **Role definitions**: `shared/schema.ts` exports `DISCORD_ROLES`, `CATEGORY_ROLE_ACCESS`, and `CATEGORIES` constants

### Categories
- Atlantis Guidelines, New Members, Internal Affairs, Foreign Affairs, Military, Economy, Technology, First Steps, Intelligence

### Storage Layer
- `server/storage.ts` — `DatabaseStorage` class implementing `IStorage` interface for document CRUD
- Uses the shared Drizzle `db` instance from `server/db.ts`

### Build System
- **Dev**: `npm run dev` — runs `tsx server/index.ts` with Vite middleware
- **Build**: `npm run build` — runs `script/build.ts` which builds client with Vite and server with esbuild
- **Production**: `npm start` — runs `node dist/index.cjs`
- **Type Check**: `npm run check` — runs TypeScript compiler in noEmit mode

### Image Uploads
- **Endpoint**: POST `/api/upload-image` (admin only, multipart form with "image" field, max 10MB)
- **Storage**: Images stored in `./uploads/` directory, served statically at `/uploads/`
- **Supported formats**: JPEG, PNG, GIF, WebP, SVG
- **Editor integration**: TipTap Image extension with paste, drag-and-drop, and toolbar upload support
- **Sanitization**: DOMPurify configured to allow `<img>` tags with src, alt, width, height, style attributes

### Discord Bot
- **File**: `server/discord-bot.ts` — Discord bot with `/atlantis` slash command
- **Startup**: Bot starts automatically when the web server starts (called from `server/index.ts`)
- **Slash Command**: `/atlantis` — replies with an embed linking to the Atlantis Archive website
- **Library**: discord.js
- **Registration**: Command registered as a guild command (using DISCORD_GUILD_ID) for instant availability

### Landing Page
- **Background**: Generated underwater Atlantis fantasy city image (`client/src/assets/images/atlantis-bg.png`)
- **Dark wash overlay**: Multiple gradient layers ensure text readability over the background image

## Discord Setup
- **Redirect URL**: Must be configured in Discord Developer Portal → OAuth2 → Redirects: `https://<your-domain>/api/callback`
- **Bot**: Must be added to the Atlantis Discord server with permissions to read guild members and roles
- **Required Bot Permissions**: Read Members, Manage Roles (read), Use Slash Commands
